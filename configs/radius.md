# RADIUS 기반 중앙 인증 시스템 구축

## 1. 초기 구축

FreeRADIUS를 설치하고 네트워크 장비와 연동하여  
인증(Authentication) 및 권한(Authorization) 테스트를 진행하였다.

초기 테스트 결과 모든 장비에서 정상적으로 동작하였다.

---

## 2. 보안 이슈 확인

RADIUS 설정 과정에서  
사용자 인증 정보가 설정 파일에 평문으로 저장되는 이슈를 확인하였다.

- 계정 정보 노출 위험
- 보안 정책에 부적합

---

## 3. 개선 작업 (DB 연동)

보안 강화를 위해 인증 구조를 DB 기반으로 변경하였다.

- 사용자 계정 DB 저장
- 네트워크 장비(Client) 정보 DB 등록
- DB 접근 권한 제한 (관리자만 접근)

---

## 4. 문제 발생

---

### 4.1.1 - DB 연동 후 테스트 진행 결과

- 일부 장비: 정상 동작
- L2 스위치: 로그인은 가능하지만 권한 적용 실패

### 4.1.2 - 해결 과정

- 원인
  인증은 성공하지만 Authorization이 정상적으로 동작하지 않는 문제 발생

- 조치
  문제 해결을 위해 VTY 라인에 권한을 강제로 적용

```bash
line vty 0 4
 privilege level 15
```

### 4.1.3 - 결과
RADIUS 기반 중앙 인증 시스템 구축
DB 기반 계정 관리 구조 적용
장비별 동작 차이를 고려한 문제 해결 경험 확보

---

### 4.2.1 - RADIUS DB 암호화 인증 문제
  
- RADIUS 사용자 정보를 DB에 저장하는 과정에서  
  보안을 위해 비밀번호를 암호화하여 저장한 이후 인증이 실패하는 문제가 발생

### 4.2.2 - 해결 과정

- 원인  
  FreeRADIUS는 기본적으로 입력된 비밀번호와 저장된 값을 비교하는 방식인데,  
  DB에 저장된 암호화 방식과 RADIUS의 인증 방식이 일치하지 않아  
  정상적인 비교가 이루어지지 않음

- 조치  
  SHA-512 기반 해시 방식($6$)으로 비밀번호를 생성  
  RADIUS에서 해당 해시를 인식할 수 있도록 설정 적용

### 4.2.3 - 결과  
  암호화된 비밀번호 기반으로도 정상적인 인증이 가능하도록 구성하였으며,
  보안성과 인증 기능을 동시에 확보

---

## 5. 배운 점
인증(Authentication)과 권한(Authorization)은 별도로 검증 필요
장비별 권한 처리 방식 차이 존재
보안 개선 과정에서 새로운 문제가 발생할 수 있음
