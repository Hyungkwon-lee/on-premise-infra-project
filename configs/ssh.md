# SSH 접근 통제 및 보안 설정

## 1. 구축 목적

서버 접근 시 보안 강화를 위해  
기본적인 패스워드 인증 방식 대신  
보안 설정을 적용하여 접근 통제 구성

---

## 2. 문제 인식

기본 SSH 설정의 경우:

- 패스워드 기반 로그인 → 보안 취약
- root 계정 직접 접속 가능
- 무차별 대입 공격(Brute Force)에 취약

---

## 3. 보안 설정 적용

### SSH Key 기반 인증 적용
- 공개키 / 개인키 기반 인증 방식 적용

---

### root 로그인 제한
```bash
PermitRootLogin no
AllowUsers user1
```
---

## 4. 결과
비밀번호 기반 공격 차단
인증 보안 수준 향상
서버 접근 통제 강화

---

## 5. 배운 점
기본 설정 그대로 사용하는 것은 보안상 위험
SSH는 최소한의 보안 설정이 필수
인증 방식 변경이 보안 수준에 큰 영향을 줌
